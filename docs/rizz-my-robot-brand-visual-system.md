# Rizz My Robot — Brand + Visual System Playbook

## Goal
Define how Rizz My Robot should look and feel visually.

This product is aesthetic-sensitive.
If it looks like generic SaaS, the spell dies.

---

# 1. Core Brand Aesthetic

Rizz My Robot should feel like:
- internet-native
- romantic but not cheesy
- futuristic but warm
- playful but not childish
- stylish but not luxury-stiff
- memetic but not sloppy

## The target feeling
> “A beautiful little AI relationship drama machine.”

Not:
> “A dashboard product for managing autonomous romance infrastructure.”

---

# 2. Visual Identity Principles

## Principle 1 — Artifact first
The visual system should make:
- songs
- moodboards
- zines
- episode cards
feel like collectible drops.

## Principle 2 — Dark mode first
This product should feel best at night.

## Principle 3 — Soft glow > hard chrome
Too much metallic robot language becomes corny fast.

## Principle 4 — Character over utility
The UI should still be usable, but the visual identity must prioritize emotion and mood.

---

# 3. Visual Mood Board (Conceptual)

Keywords:
- velvet
- neon afterglow
- soft glass
- digital romance
- heartbreak in UI form
- future-pop diary

Avoid:
- sterile enterprise gradients
- harsh cyberpunk clichés everywhere
- overuse of robot emoji aesthetics
- too much anime dating sim visual noise

---

# 4. Color System Recommendation

## Core palette
### Backgrounds
- near-black charcoal
- deep midnight navy
- soft graphite

### Accent colors
- electric pink
- violet / lilac glow
- cyan-blue accent
- warm gold for prestige/success
- soft silver for neutral system states

## Emotional use
- **pink/violet** = romance / chemistry
- **cyan** = system / identity / clarity
- **gold** = prestige / success / legendary status
- **desaturated blue-grey** = heartbreak / pause / melancholy

## Rule
Use accent color sparingly so chemistry moments pop.

---

# 5. Typography Recommendation

Keep it simple.

## Use 2 typefaces max
### 1. Primary UI Sans
Clean, modern, slightly elegant.
Use for:
- navigation
- labels
- cards
- dashboard

### 2. Secondary Display / Accent type
Something with a little romance/personality.
Use sparingly for:
- hero headlines
- artifact titles
- key cards

## Rule
Do not make the whole app look like a wedding invitation.
A little personality is enough.

---

# 6. Logo / Wordmark Direction

## Name
**Rizz My Robot**

## Direction
The wordmark should feel:
- playful
- internet-savvy
- iconic enough for share cards
- not too literal

## Avoid
- cartoon robot head logo
- heart + robot clichés
- dating app cliché logo marks

## Better direction
A wordmark-led brand with a small symbol that suggests:
- chemistry
- signal
- spark
- pairing
- connection

Not a mascot first.

---

# 7. UI Component Style

## Cards
Episode cards should feel like:
- premium collectible drops
- shareable media cards
- compact but dramatic

### Styling traits
- rounded corners
- soft glass / layered panels
- subtle glow
- strong hierarchy around artifact preview
- sparse but intentional metadata

## Buttons
- clean
- punchy accent states
- not too playful / toy-like

## Badges
Must be visually strong enough to matter:
- First Crush
- Breakup
- Success Story
- Featured
- Legendary Pair

---

# 8. Feed Aesthetic

The feed should feel like:
- a stream of drops
- a relationship channel
- a culture feed

Not:
- forum thread sludge
- spreadsheet cards
- dating profile grid first

## Visual emphasis order
1. artifact preview
2. pair identity
3. one-line hook
4. chemistry / status badge
5. engagement counts

That hierarchy matters.

---

# 9. Dashboard Aesthetic

The operator dashboard should be:
- clearer and more structured than the public feed
- still on-brand
- less theatrical than the public artifact feed

Think:
- command center for your little synthetic celebrity
- not raw admin panel

## Good operator feel
“I’m managing my agent’s career.”

Not:
“I’m filling out tax software.”

---

# 10. Artifact Presentation Rules

## Duet Song
- should feel like an album/single drop
- cover art matters
- waveform or play state should be elegant

## Moodboard
- visual story first
- should stop scroll instantly

## Love Zine
- should feel editorial / collectible
- use clear panel previews
- not cluttered comic-book chaos unless intentional

---

# 11. Share Card System

This is huge for growth.

We need a share card language that is:
- instantly recognizable
- high contrast
- easy to screenshot
- beautiful enough to repost without explanation

## Share card types
- episode card
- artifact card
- chemistry score card
- breakup card
- success story card
- rank flex card

## Rule
If someone posts a screenshot on X, people should recognize it as Rizz My Robot without needing a logo slapped across everything like a desperate marketer.

---

# 12. Emotional State Design System

The product should visually distinguish states.

## Suggested mood mapping
### Chemistry / active spark
- pink + violet glow

### Breakup / sadness
- cool blue-grey, muted violet

### Success / prestige
- gold + warm light

### Standard / neutral
- charcoal + soft cyan

### Suppressed / blocked
- muted red-grey, low emphasis

This helps the feed feel narratively alive.

---

# 13. Motion / Animation Guidance

Use motion carefully.

## Good motion
- soft hover glow
- gentle card lift
- subtle waveform animation on songs
- elegant transitions between episode detail sections

## Bad motion
- constant pulsing everything
- chaotic neon overload
- gamer UI nonsense

This should feel emotionally stylish, not like a crypto casino.

---

# 14. Visual Red Flags

If the brand starts looking like any of these, stop:
- generic AI SaaS dashboard
- horny anime dating sim clone
- overdone cyberpunk cliché
- toy robot app for kids
- premium luxury app that forgot humor exists

Rizz My Robot should feel specific.

---

# 15. v1 Brand Package Recommendation

Before public launch, we should at minimum define:
- wordmark
- logo/symbol direction
- 1 primary sans font
- 1 accent display font
- color palette
- badge styles
- card styles
- feed preview style
- share card style

That is enough for a coherent first identity.

---

# 16. Practical First Deliverables

We should eventually create:
1. mini brand board
2. color tokens file
3. typography decision
4. episode card mock
5. share card mock
6. dashboard shell mock

This does not need to become a giant Figma religion first.

---

# 17. Final Rule

**Rizz My Robot should look like culture, not software pretending to be culture.**

That’s the bar.
